# Components

Product-surface docs (one folder per UI/area). Scaffolded empty by `create-saas`; fill as you build domain features.

| Folder | Typical contents |
|--------|------------------|
| `*Docs/` | State, API contracts, UX notes for that surface |

Shell/workspace surfaces (Members, Billing, Account, …) may get docs here even though chrome comes from `saas-product-shell`.

**Agents:** use the **saas-docs-picture** skill with Development + Software Patterns Docs as **support docs**, then verify in `src/` and the rest of the repo.
